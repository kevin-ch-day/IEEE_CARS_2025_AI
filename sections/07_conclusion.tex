\section{Conclusion and Future Work}

This paper presented a fixed-cutoff evidence model for assessing privacy and security risk in Android consumer applications using static and runtime evidence. At cutoff, 15/15 Research Dataset Beta apps had paper-usable, build-backed evidence bundles, and the cutoff contained zero true evidence holes. The key result is not that every live installed app was complete on the current build; the key result is that every app had auditable evidence with preserved build, static, dynamic, APK, PCAP, and QA provenance.

The study integrates static and dynamic evidence across an expanded 15-app cohort. Static analysis supplies declared and embedded risk context, while runtime analysis supplies observed baseline, QFG, and interactive behavior. The cutoff model connects these surfaces through explicit evidence tiers so the paper can distinguish strict current-build completion, current-build mixed baseline evidence, and retained prior-build paper evidence.

Future work should use the frozen cutoff as a comparison point for refresh waves, additional devices, newer Android versions, and longitudinal update analysis. For the present paper, the evidence boundary supports writing from the final cutoff bundle while preserving the caveats needed for reproducible interpretation.
