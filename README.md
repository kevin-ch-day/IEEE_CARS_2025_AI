# IEEE CARS 2026 Android Static/Runtime Evidence Paper

This repository contains an IEEE conference-paper draft in the ScytaleDroid Android privacy/security research line. The paper is intended for [IEEE CARS 2026](https://www.ieee-cars.org/) and focuses on an empirical 15-app Android consumer-app study using static and runtime evidence.

The project is scoped to defensive, non-invasive Android application analysis. It uses build-backed evidence bundles, static analysis results, dynamic evidence packs, packet-capture availability, and quality-assurance provenance generated by ScytaleDroid.

Author pages and conference rules were cross-checked against:
- [IEEE CARS 2026 authors and formatting guidance](https://www.ieee-cars.org/authors)
- [IEEE CARS call for papers](https://www.ieee-cars.org/call-for-papers)

## Repository layout

- `main.tex` — IEEE conference paper driver file (`\documentclass[conference]{IEEEtran}`)
- `preliminary/` — title, authors, abstract, and IEEE keywords:
  - `00_paper_header.tex`
  - `authors.tex`
  - `abstract.tex`
- `content/` — paper body split into modular files:
  - `01_introduction.tex`
  - `02_background.tex`
  - `03_lit_review.tex`
  - `04_methodology.tex`
  - `05_results.tex`
  - `06_discussion.tex`
  - `07_conclusion.tex`
- `references/references.bib` — bibliography file
- `figures/` — publication figures generated from ScytaleDroid report outputs
- `tables/` — publication-ready LaTeX tables and matching CSV exports
- `generated/source_data/` — source CSV/JSON inputs and checksum manifests used to build tables and figures
- `templates/` — IEEE template, sample figure, and reference files
- `scripts/` — local build helpers
- `.gitignore` and `.gitattributes` — clean source control and line ending normalization
- `IEEEtran.cls` — conference class file at repo root

## Build instructions

Build with BibTeX through `latexmk`:

```bash
latexmk -pdf -interaction=nonstopmode -halt-on-error main.tex
```

Manual equivalent:

```bash
pdflatex main.tex
bibtex main
pdflatex main.tex
pdflatex main.tex
```

Quick build script, if running from PowerShell:

```bash
./scripts/build_paper.ps1
```

Clean generated files with:

```bash
latexmk -C
```

Regenerate publication tables and figures from the current ScytaleDroid alignment bundle with:

```bash
python scripts/generate_publication_assets.py
```

## Research scope and safety posture

The intended framing is:

- Android consumer-app privacy and security risk assessment
- static and runtime evidence integration
- build/version-backed evidence bundles
- fixed cutoff methodology for app-update churn
- strict idle, quiescent foreground, and interactive runtime classes
- limitations, provenance, and ethics

The draft is intentionally paper-first and defensive: clear evidence boundaries, controlled non-root collection, and explicit limitations.
