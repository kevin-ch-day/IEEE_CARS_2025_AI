\section{Discussion and Limitations}

The 14-day window is useful because it separates paper evidence readiness from the live operational queue. If a selected build was captured inside the window and linked to static and runtime evidence, a later app-store update does not erase the evidence. It only creates a future longitudinal comparison point. This framing prevents valid recent measurements from being discarded solely because the installed app changed after capture.

Static and runtime evidence also diverge in expected ways. Static exposure indicates declared capability, packaged attack surface, and detector posture. Runtime evidence indicates what appeared during selected execution windows. A high static count does not imply high runtime activity, and low observed runtime activity does not prove low privacy risk. The integrated results should therefore be interpreted as cross-layer characterization, not causation or prediction.

QFG separation is a practical methodological contribution. Social, news, and messaging apps may continue loading content or maintaining state while foregrounded and untouched. Counting those captures as strict idle would overstate baseline quietness; discarding them would lose meaningful app-driven behavior. Reporting QFG separately keeps both interpretations visible.

The study is bounded by a 15-app dataset, one Android 15 physical device, selected user scenarios, available static detectors, and a non-root no-MITM PCAP privacy model. It does not claim exhaustive code coverage, decrypted payload semantics, or full Data Safety validation. Some apps have stronger interactive depth than others, and runtime findings are scenario-bounded. Static detector counts also remain descriptive exposure indicators rather than formal exploitability or compliance scores.

Future work should add longitudinal build-to-build comparisons, broader device and Android-version coverage, stronger service-family labeling, and controlled held-out runtime scenarios. The current contribution is a reproducible, build-backed static/runtime evidence model for recent Android consumer-app measurements.
