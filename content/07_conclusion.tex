\section{Conclusion and Future Work}

This paper presented a cutoff-window evidence model for assessing privacy and security risk in Android consumer applications using static and runtime evidence. At cutoff, all 15 apps had recent, build-backed evidence bundles, and the cutoff contained zero true evidence holes. The key result is that every app had auditable evidence with preserved build, static, dynamic, APK, PCAP, collection-window, and QA provenance.

The study integrates static and dynamic evidence across a 15-app dataset. Static analysis supplies declared and embedded risk context, while runtime analysis supplies observed baseline, QFG, and interactive behavior. The cutoff-window model connects these surfaces through selected build-backed evidence bundles so the paper can report recent app evidence without treating normal app-store churn as evidence loss.

Future work should use the frozen cutoff as a comparison point for refresh waves, additional devices, newer Android versions, and longitudinal update analysis. For the present paper, the evidence boundary supports writing from the final cutoff bundle while preserving the caveats needed for reproducible interpretation.
