\section{Conclusion}

This paper presented a cutoff-window evidence model for assessing privacy and security risk in Android consumer applications using static and runtime evidence. At cutoff, all 15 apps had recent, build-backed evidence bundles with zero true evidence holes, preserving build, static, dynamic, APK, PCAP, collection-window, and QA provenance. Static analysis supplies declared and embedded risk context, while runtime analysis supplies observed baseline, QFG, and interactive behavior; the cutoff-window model connects these surfaces without treating normal app-store churn as evidence loss.

Future work will extend the frozen cutoff through longitudinal build-to-build comparisons, additional devices and Android versions, improved service-family labeling, and controlled held-out runtime scenarios. The present study establishes a reproducible, build-backed static-runtime evidence model for recent Android consumer-app measurements while preserving the provenance and limitations needed for reliable interpretation.
